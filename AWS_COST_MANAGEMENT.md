# AWS Amplify コスト管理ガイド

## 📊 コスト概算

### AWS Amplify ホスティング料金

AWS Amplifyのホスティングは**無料枠**が非常に充実しています：

| 項目 | 無料枠 | 超過時の料金 |
|------|-------|------------|
| **ビルド時間** | 1,000分/月 | $0.01/分（約1.5円/分） |
| **ホスティング** | 15GB ストレージ<br>100GB 転送量/月 | ストレージ: $0.023/GB/月<br>転送: $0.15/GB |
| **プレビュー環境** | 1,000分/月 | $0.01/分 |

### 予想される月額コスト（小規模プロジェクト）

**通常の使用パターン（月10回デプロイ、1回5分）の場合：**

```
ビルド時間: 10回 × 5分 = 50分/月
→ 無料枠内（1,000分/月）✅

ホスティング: 
- ストレージ: 約100MB（無料枠内）✅
- 転送量: 約10GB/月（無料枠内）✅

**月額コスト: $0（無料）** 🎉
```

### 予想される月額コスト（中規模プロジェクト）

**活発な開発（月50回デプロイ、1回5分）の場合：**

```
ビルド時間: 50回 × 5分 = 250分/月
→ 無料枠内（1,000分/月）✅

ホスティング: 
- ストレージ: 約500MB（無料枠内）✅
- 転送量: 約50GB/月（無料枠内）✅

**月額コスト: $0（無料）** 🎉
```

### 予想される月額コスト（大規模プロジェクト）

**非常に活発な開発（月200回デプロイ、1回5分）の場合：**

```
ビルド時間: 200回 × 5分 = 1,000分/月
→ 無料枠内（1,000分/月）✅

ホスティング: 
- ストレージ: 約2GB（無料枠内）✅
- 転送量: 約100GB/月（無料枠内）✅

**月額コスト: $0（無料）** 🎉
```

### ⚠️ コストが発生する可能性がある場合

**月1,000分を超えるビルド時間の場合：**

例：月300回デプロイ、1回5分 = 1,500分/月
```
超過分: 1,500 - 1,000 = 500分
コスト: 500分 × $0.01 = $5/月（約750円/月）
```

**月100GBを超える転送量の場合：**

例：月150GBの転送
```
超過分: 150 - 100 = 50GB
コスト: 50GB × $0.15 = $7.50/月（約1,125円/月）
```

## 🛡️ 予期しない高額請求を防ぐ設定

### 1. AWS Budgets（予算アラート）の設定

**重要**: 予算アラートを設定して、予期しない請求を防ぎます。

#### 設定手順

1. [AWS Billing Console](https://console.aws.amazon.com/billing/)にアクセス
2. 左メニュー → **「Budgets」**をクリック
3. **「Create budget」**をクリック
4. 予算タイプを選択：
   - **「Cost budget」**を選択
5. 予算の詳細を設定：
   - **予算名**: `BASHOTORI-Amplify-Monthly`
   - **期間**: `Monthly`
   - **予算額**: `$10`（約1,500円/月）
   - **予算の開始日**: 今月の1日
6. アラートの設定：
   - **「Alert threshold」**: `80%`（$8でアラート）
   - **「Alert threshold」**: `100%`（$10でアラート）
   - **「Alert threshold」**: `120%`（$12でアラート）
   - **通知先**: メールアドレスを入力
7. **「Create budget」**をクリック

#### 推奨設定値

| 予算額 | アラート閾値 | 用途 |
|--------|------------|------|
| $10/月 | 80%, 100%, 120% | 小規模プロジェクト（通常は無料） |
| $50/月 | 80%, 100%, 120% | 中規模プロジェクト |
| $100/月 | 80%, 100%, 120% | 大規模プロジェクト |

### 2. Amplify ビルド設定の最適化

#### 2.1 ビルド時間の短縮

`amplify.yml`を最適化して、ビルド時間を短縮します：

```yaml
version: 1
frontend:
  phases:
    preBuild:
      commands:
        - npm ci --prefer-offline --no-audit  # キャッシュを活用
    build:
      commands:
        - npm run build
  artifacts:
    baseDirectory: dist
    files:
      - '**/*'
  cache:
    paths:
      - node_modules/**/*
```

#### 2.2 不要なブランチの自動デプロイを無効化

1. AWS Amplify Console → アプリ「BASHOTORI」を選択
2. 左メニュー → **「ホスティング」** → **「ビルドの設定」**
3. 各ブランチの設定を確認：
   - **本番ブランチ（main）**: 自動デプロイ有効 ✅
   - **開発ブランチ（developなど）**: 必要に応じて無効化
   - **プレビュー環境**: 必要に応じて無効化

### 3. CloudWatch アラームの設定

#### 3.1 ビルド時間の監視

1. [CloudWatch Console](https://console.aws.amazon.com/cloudwatch/)にアクセス
2. 左メニュー → **「アラーム」** → **「アラームの作成」**
3. メトリクスを選択：
   - **名前空間**: `AWS/Amplify`
   - **メトリクス名**: `BuildDuration`
4. 条件を設定：
   - **閾値**: `600秒`（10分）
   - **アラーム条件**: `>= 600`
5. 通知先を設定（SNSトピックまたはメール）
6. **「アラームの作成」**をクリック

#### 3.2 転送量の監視

1. CloudWatch Console → **「アラーム」** → **「アラームの作成」**
2. メトリクスを選択：
   - **名前空間**: `AWS/Amplify`
   - **メトリクス名**: `BytesDownloaded`
3. 条件を設定：
   - **統計**: `Sum`
   - **期間**: `1日`
   - **閾値**: `100GB`（無料枠の上限）
   - **アラーム条件**: `>= 100GB`
4. 通知先を設定
5. **「アラームの作成」**をクリック

### 4. IAM ユーザーの権限制限

#### 4.1 最小権限の原則

Amplify用のIAMユーザーには、必要最小限の権限のみを付与します：

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "amplify:*"
      ],
      "Resource": "arn:aws:amplify:ap-northeast-1:*:apps/d31rs0a0flg3u3/*"
    },
    {
      "Effect": "Deny",
      "Action": [
        "amplify:DeleteApp",
        "amplify:DeleteBranch"
      ],
      "Resource": "*"
    }
  ]
}
```

### 5. 自動デプロイの制限

#### 5.1 プルリクエストの自動デプロイを無効化（オプション）

1. AWS Amplify Console → アプリ「BASHOTORI」を選択
2. 左メニュー → **「ホスティング」** → **「プレビュー」**
3. **「プルリクエストのプレビューを無効化」**を選択（必要に応じて）

#### 5.2 ビルドの手動承認を有効化（オプション）

1. AWS Amplify Console → アプリ「BASHOTORI」を選択
2. 左メニュー → **「ホスティング」** → **「ビルドの設定」**
3. **「ビルドの手動承認を有効化」**を選択（必要に応じて）

### 6. 定期的なコスト確認

#### 6.1 月次コストレポートの確認

1. [AWS Billing Console](https://console.aws.amazon.com/billing/)にアクセス
2. 左メニュー → **「コストと使用量」** → **「コストエクスプローラー」**
3. サービス別のコストを確認：
   - **AWS Amplify**のコストを確認
   - 異常な増加がないか確認

#### 6.2 使用量の監視

1. AWS Amplify Console → アプリ「BASHOTORI」を選択
2. 左メニュー → **「概要」**
3. **「使用量」**セクションで以下を確認：
   - ビルド時間の使用量
   - 転送量の使用量
   - ストレージの使用量

## 🚨 緊急時の対応

### 予期しない高額請求が発生した場合

1. **即座にAmplifyアプリを一時停止**
   - AWS Amplify Console → アプリ「BASHOTORI」を選択
   - **「アプリの削除」**または**「ブランチの削除」**を実行

2. **コストの原因を特定**
   - AWS Billing Console → コストエクスプローラー
   - サービス別のコストを確認
   - 異常な使用量がないか確認

3. **AWSサポートに連絡**
   - [AWS Support Center](https://console.aws.amazon.com/support/)にアクセス
   - 予期しない請求について問い合わせ

## 💡 コスト削減のヒント

### 1. ビルド時間の短縮

- `npm ci`の代わりに`npm install`を使用（開発環境のみ）
- 不要な依存関係を削除
- ビルドキャッシュを活用

### 2. 転送量の削減

- 画像の最適化（既に実装済み）
- 静的アセットのCDNキャッシュを活用
- 不要なファイルをビルドから除外

### 3. 不要なリソースの削除

- 使用していないブランチを削除
- 古いビルドログを削除
- プレビュー環境を必要に応じて無効化
